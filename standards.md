# Standards implemented

This project implements the following [standards](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards)
from the crzynet `homelab-configs` repo. Each row pins the **version** that this
project has actually wired up.

| Standard | Version | Adopted | Notes |
|---|---|---|---|
| [vexp-context-engine](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/vexp-context-engine/README.md) | 3.0.0 (de-adopted) | 2026-06-01 | **De-adopted / sunset 2026-06-01.** vexp is removed homelab-wide; the standard is rewritten as a removal guide at **v3.0.0**. All repo wiring stripped (was fully adopted at v2.1.0, this repo being the standard's first adopter): the `vexp-guard.sh` PreToolUse hook, the `mcp__vexp__*` allows + hook entry in `.claude/settings.json` (sandbox block preserved), the Context-search snippet in `CLAUDE.md`, `.vexpignore`, the tracked `.vexp/` files, and the gitignored auto-generated `.claude/CLAUDE.md`. Host daemon teardown is **not** done from this repo — it's the Ansible `devworkstation` role's opt-in `--tags vexp_teardown` (a running daemon may transiently recreate the untracked `.vexp/` runtime dir until then). See the v3.0.0 removal guide. |
| [code-checkin-and-pr](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/code-checkin-and-pr/README.md) | 1.2.0 | 2026-06-15 | **Wired.** Adopted at v1.1.0 (5 required checks) on 2026-05-28; **re-pinned to v1.2.0 on 2026-06-15** when the standard added the test + SAST checks. All 7 required checks live across `.github/workflows/`: in `ci.yml` (workflow `CI`) — Lint (`ruff` + `tsc --noEmit`), Config validation (YAML/JSON parse), Migration check (`alembic upgrade head` → assert `(head)`), Compose validation (`docker compose config --quiet`), Image build (PR-only, no push), and **Test (`pytest`, now a required check, not a bonus job)**; in `codeql.yml` (workflow `CodeQL`, action pinned `v4`) — **SAST via CodeQL `Analyze (python)` + `Analyze (javascript-typescript)`** on push to `dev`/`main`, PRs to `main`, and weekly. Publish matrix in `publish.yml` (workflow `Build and publish Docker images`): push→`dev` ⇒ `:dev`+`:sha-*`; push→`main` ⇒ `:latest`+`:sha-*`; release published ⇒ `:latest`+`:<semver>`+`:<major>`. Registry: `ghcr.io/crzykidd/filament-bridge`. Retention in `retention.yml`: `:sha-*` keep 30, semver keep 15, protected tags never pruned. `main` branch protection applied — **8 required status contexts** (the 6 `CI / *` jobs + the 2 `CodeQL / Analyze *` jobs) + PR required, admin bypass kept; see `docs/decisions.md` for the runbook. One historical slip: commit `1d95a02` carries a forbidden `Co-authored-by:` trailer (acknowledged, not corrected retroactively). |
| [release-prep-and-cut](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/release-prep-and-cut/README.md) | 1.0.0 | 2026-05-28 | **Adopted; operational.** Slash commands at `.claude/commands/release-prep.md` + `release-cut.md`, snippet in `CLAUDE.md`. Release artifacts exist: canonical version file `backend/app/__init__.py` (`__version__ = "0.2.0"`), `CHANGELOG.md` (Keep-a-Changelog, `[Unreleased]` seeded), README version badge and "What's New" section. The publish-on-`release` workflow (`.github/workflows/publish.yml`, triggered by `release: published`) is now wired — first release is unblocked. |
| [handoff-prompt-workflow](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/handoff-prompt-workflow/README.md) | 1.5.0 | 2026-05-28 | `prompts/TEMPLATE.md` and `docs/decisions.md` created.
