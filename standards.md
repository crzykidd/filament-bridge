# Standards implemented

This project implements the following [standards](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards)
from the crzynet `homelab-configs` repo. Each row pins the **version** that this
project has actually wired up.

| Standard | Version | Adopted | Notes |
|---|---|---|---|
| [vexp-context-engine](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/vexp-context-engine/README.md) | 3.0.0 (de-adopted) | 2026-06-01 | **De-adopted / sunset 2026-06-01.** vexp is removed homelab-wide; the standard is rewritten as a removal guide at **v3.0.0**. All repo wiring stripped (was fully adopted at v2.1.0, this repo being the standard's first adopter): the `vexp-guard.sh` PreToolUse hook, the `mcp__vexp__*` allows + hook entry in `.claude/settings.json` (sandbox block preserved), the Context-search snippet in `CLAUDE.md`, `.vexpignore`, the tracked `.vexp/` files, and the gitignored auto-generated `.claude/CLAUDE.md`. Host daemon teardown is **not** done from this repo — it's the Ansible `devworkstation` role's opt-in `--tags vexp_teardown` (a running daemon may transiently recreate the untracked `.vexp/` runtime dir until then). See the v3.0.0 removal guide. |
| [code-checkin-and-pr](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/code-checkin-and-pr/README.md) | 1.1.0 | 2026-05-28 | **Partial.** Operational-rules snippet in `CLAUDE.md`; `dev` + `main` branches exist; commits follow the conventional prefixes. The full stack now exists (`backend/`, `frontend/`, `Dockerfile`, `docker-compose.yml`, `backend/alembic/`), so the "pending first code" deferral is over: the 5 required CI checks (backend lint `ruff`, structured-config validation, alembic migration-to-head, `docker compose config`, image build) and `main` branch protection are **not yet wired** (no `.github/workflows/`), nor is the image-publish matrix / registry retention. Tracked for wiring in `prompts/2026-06-02-ci-checkin-workflows.md`. One historical slip: commit `1d95a02` carries a forbidden `Co-authored-by:` trailer. |
| [release-prep-and-cut](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/release-prep-and-cut/README.md) | 1.0.0 | 2026-05-28 | **Adopted; not yet operational.** Slash commands at `.claude/commands/release-prep.md` + `release-cut.md`, snippet in `CLAUDE.md`. The release artifacts now exist: canonical version file `backend/app/__init__.py` (`__version__ = "0.1.0"`), `CHANGELOG.md` (Keep-a-Changelog, `[Unreleased]` seeded), the README version badge, and the README "What's New" section. Remaining blocker to a first release: the publish-on-`release` CI workflow (shared with `code-checkin-and-pr`, not yet wired). |
| [handoff-prompt-workflow](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/handoff-prompt-workflow/README.md) | 1.5.0 | 2026-05-28 | `prompts/TEMPLATE.md` and `docs/decisions.md` created. Soft pointer in `CLAUDE.md`. v1.5.0 adds step 8: the agent hands the user the launch command (`claude --model <model> "Read prompts/<file>.md and execute it as your task."`) whenever it creates a prompt. `TEMPLATE.md` unchanged from 1.4.0. |
| [repo-sandbox-permissions](https://gitea.crzynet.com/crzynet/homelab-configs/src/branch/main/standards/repo-sandbox-permissions/README.md) | 1.0.0 | 2026-05-30 | **scope: repo-wide (`.claude/settings.json`), 2026-05-30.** Canonical sandbox block: `sandbox.enabled` + `autoAllowBashIfSandboxed`, `Read/Edit/Write(**)` allows. (Originally merged alongside the vexp MCP allows + guard hook; those were removed when vexp was de-adopted 2026-06-01 — the sandbox block was preserved intact.) Per-project blanks filled for the Python+Node stack: `network.allowedDomains` = `localhost`, `127.0.0.1`, `pypi.org`, `files.pythonhosted.org`, `registry.npmjs.org`; `filesystem.allowWrite` = `/tmp/**`, `~/.cache/**`. No `CLAUDE-snippet.md` (map-not-copy link-back); `CLAUDE.md` carries a pointer. |
